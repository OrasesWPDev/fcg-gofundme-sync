# Phase 9.1: Environment-Safe Configuration

## Goal

Enable environment-specific credentials via wp-config.php constants with hostname detection, ensuring database copies between staging and production don't cause credential cross-contamination.

## Background

**Problem:** When copying production database to staging on WP Engine:
- wp_options values (campaign ID, component ID) get overwritten
- Risk of staging using production API credentials or vice versa

**Solution:** Move all environment-specific values to wp-config.php with hostname-based detection. Both sets of credentials live in the file, but the correct ones are selected based on the current hostname.

## Scope

### In Scope
1. Add constant support for `GOFUNDME_MASTER_CAMPAIGN_ID` and `GOFUNDME_MASTER_COMPONENT_ID`
2. Update admin UI to show read-only values when constants are defined
3. Simplify admin UI (remove editable ID fields when using constants)
4. Keep polling controls (Auto-Polling toggle, Polling Interval)
5. Keep Sync Status section (Last Poll, Sync All Now button)
6. Keep Recent Conflicts table
7. Document wp-config.php setup

### Out of Scope
- OAuth credential changes (already support constants)
- GitHub Actions deployment (simpler solution chosen)
- Local development environment setup

### Task 1: Add Constant Support in Plugin

**File:** `includes/class-admin-ui.php`

Update `get_master_campaign_id()` and `get_master_component_id()` methods (or add them):

```php
private function get_master_campaign_id() {
    // Constants take priority over wp_options
    if (defined('GOFUNDME_MASTER_CAMPAIGN_ID') && GOFUNDME_MASTER_CAMPAIGN_ID) {
        return GOFUNDME_MASTER_CAMPAIGN_ID;
    }
    return get_option('gofundme_master_campaign_id', '');
}

private function get_master_component_id() {
    if (defined('GOFUNDME_MASTER_COMPONENT_ID') && GOFUNDME_MASTER_COMPONENT_ID) {
        return GOFUNDME_MASTER_COMPONENT_ID;
    }
    return get_option('gofundme_master_component_id', '');
}

private function is_config_from_constants() {
    return defined('GOFUNDME_MASTER_CAMPAIGN_ID') && GOFUNDME_MASTER_CAMPAIGN_ID;
}
```

**File:** `includes/class-sync-handler.php`

### Task 4: Document wp-config.php Setup

**File:** `docs/environment-configuration.md`

```markdown
# Environment-Safe Configuration

## Overview

To prevent credential cross-contamination when copying databases between
environments (e.g., production → staging), all Classy API credentials
should be configured in wp-config.php using hostname detection.

## Setup

Add this block to wp-config.php on BOTH staging and production:

\`\`\`php
/**
 * FCG GoFundMe Sync - Environment-Specific Configuration
 */
if (strpos($_SERVER['HTTP_HOST'] ?? '', 'frederickc2stg') !== false) {
    // STAGING (Sandbox API)
    define('GOFUNDME_CLIENT_ID', 'sandbox_client_id');
    define('GOFUNDME_CLIENT_SECRET', 'sandbox_secret');
    define('GOFUNDME_ORG_ID', 'your_org_id');
    define('GOFUNDME_MASTER_CAMPAIGN_ID', 'staging_campaign_id');
    define('GOFUNDME_MASTER_COMPONENT_ID', 'staging_component_id');
} else {
    // PRODUCTION
    define('GOFUNDME_CLIENT_ID', 'prod_client_id');
    define('GOFUNDME_CLIENT_SECRET', 'prod_secret');
    define('GOFUNDME_ORG_ID', 'your_org_id');
    define('GOFUNDME_MASTER_CAMPAIGN_ID', '764694');
    define('GOFUNDME_MASTER_COMPONENT_ID', 'prod_component_id');
}
\`\`\`

## How It Works

1. Both sets of credentials exist in wp-config.php
2. Hostname detection picks the correct set at runtime
3. Database copies don't affect credentials (they're in file, not DB)
4. Plugin reads constants with priority over wp_options

## Verification

After setup, the plugin admin page will show:
- "Configuration (from wp-config.php)" header
- Read-only display of Campaign ID and Component ID
- Info message: "Configured in wp-config.php"
```
