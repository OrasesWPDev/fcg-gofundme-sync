## Summary

Phase 9.1.1 addresses four client-reported mobile issues recorded in videos on 2026-02-19. One issue (#3, designation race condition) is already fixed and deployed to staging. The remaining three issues (#1, #2, #4) are behaviors of the Classy SDK's "nudge" system: an abandoned donation reminder overlay, a "Become a monthly supporter!" monthly upsell interstitial, and a scroll-lock that persists after the upsell flow closes on iOS Safari.

The Classy SDK has a documented "nudges" system (accessed in Embedded Studio > Settings tab) that controls the recurring donation nudge and abandoned cart nudge. These nudges are the root cause of Issues #1, #2, and #4. The key question for planning is whether these nudges are enabled in the sandbox campaign dashboard and can be toggled off self-service, or require Classy support escalation.

The theme-changes approach (tracking files to be manually applied to the WP Engine theme) is already established. Two files tracked in `theme-changes/` (functions-php-additions.php and fund-form.php) need deployment to production as part of this phase, bundled appropriately with any Classy SDK configuration changes.

**Primary recommendation:** During execution, navigate the Classy sandbox campaign dashboard to Campaign > Embedded Studio > Settings to find nudge toggles. If present, disable the recurring donation nudge and abandoned cart nudge. If not self-service, create `docs/classy-sdk-escalation.md` with the issue descriptions and video file references for Chad to send to Luke/Jon at Classy.

## What the Classy SDK Nudge System Is

Based on official GoFundMe Pro help center search results:

### Recurring Donation Nudge
- **What it does:** When donor selects a mid-range amount (not highest or lowest), prompts them to convert to monthly at 1/4 the one-time amount
- **Trigger:** Appears after amount selection before checkout completes
- **Visual:** "Become a monthly supporter!" with gift box, "Donate $X/month" and "Keep my one-time gift" buttons
- **Code parameter to enable:** `egrn: true` in the urlParams section of embed code
- **How to disable:** Remove `egrn: true` OR set `egrn: false` from embed urlParams
- **Dashboard location:** Embedded Studio > Settings tab > Nudges (can "turn on or off")
- **Confidence:** MEDIUM (search result summary from official help center; actual page returned 403)

### Abandoned Cart / Donation Reminder
- **What it does:** When user starts a donation but doesn't finish (inactivity), shows a reminder overlay
- **Visual:** "Make a difference!" overlay with "Give Now" button, appears bottom-left on desktop, full-overlay on mobile
- **Code:** A separate code snippet that must be added to website footer to enable
- **How to disable:** Remove the abandon cart nudge code snippet from website footer (if present), OR check campaign dashboard Emails tab or Donations settings for a toggle
- **Confidence:** LOW — the GoFundMe Pro help center docs are 403-blocked; can't confirm dashboard toggle location

### Scroll-Lock Side Effect
- **What it happens:** Classy SDK's checkout/upsell modal adds `overflow: hidden` or equivalent to `<body>` on iOS Safari
- **iOS Safari quirk:** When a modal opens and locks scroll, closing it does not always restore scroll on iOS — a known decade-old browser bug
- **This is NOT our code** — it's a side effect of the Classy SDK's modal system
- **Fix if not configurable:** Cannot be fixed on our end without wrapping/patching the SDK (not advisable)
- **Best resolution:** Disable the recurring donation nudge (source of the modal) via dashboard settings

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Scroll lock restoration | Custom JS that patches `overflow: hidden` on body | Disable the nudge that causes the modal | Patching Classy SDK internals is fragile and will break on SDK updates |
| Blocking Classy popups | CSS to hide Classy overlay elements | Disable via campaign dashboard | CSS hiding is fragile; Classy changes class names |
| Recreating abandon cart behavior | Custom timeout/reminder popup | Remove existing Classy one if it's the source | The "Make a difference!" popup may be a different plugin — investigate first |

**Key insight:** All three SDK issues (#1, #2, #4) should be resolved by disabling Classy-controlled features in the campaign dashboard, not by patching our code. The Classy SDK is a black box; working around its internals is unsustainable.

## Common Pitfalls

### Pitfall 1: Conflating the Two Popups
**What goes wrong:** Treating "Make a difference!" (abandoned cart) and "Become a monthly supporter!" (recurring nudge) as the same Classy feature
**Why it happens:** Both appear as overlays during the donation flow
**How to avoid:** They are separate Classy features with separate settings. Check both.
**Warning signs:** Disabling one and the other still appears

### Pitfall 2: Assuming All Three SDK Issues Share One Setting
**What goes wrong:** Finding the recurring nudge setting, disabling it, and assuming Issues #1, #2, #4 are all resolved
**Why it happens:** Issue #4 (scroll-lock) is caused by the recurring nudge modal, but Issues #1/#2 (dimming/overlay) may be caused by the abandoned cart nudge — a different setting
**How to avoid:** Test both issue types after each change. Issue #1/#2 = inactivity overlay. Issue #4 = post-amount-selection upsell.

### Pitfall 3: "Make a difference!" Popup May Not Be Classy
**What goes wrong:** Looking only in Classy campaign settings to disable the "Make a difference!" popup
**Why it happens:** Per `docs/mobile-popup-scroll-investigation.md`, the "Make a difference!" popup was NOT found in theme PHP/JS files — it may be database-stored via a WordPress plugin
**How to avoid:** The investigation doc lists candidate plugins:
  - `insert-headers-and-footers` — could inject popup JS
  - `gravityforms` — could be a lightbox
  - WordPress Customizer / Widget
  If not found in Classy dashboard, search WP database: `wp db search "Make a difference" wp_posts wp_options`
**Warning signs:** Classy dashboard has no matching setting for this popup

### Pitfall 4: Deploying fund-form.php Before Checking Staging State
**What goes wrong:** Deploying the fund-form.php from theme-changes/ to staging and accidentally overwriting recent changes
**Why it happens:** The staging fund-form.php may have been manually edited via SSH
**How to avoid:** SSH to staging first and diff the file: `cat ~/sites/frederickc2stg/wp-content/themes/community-foundation/fund-form.php` and compare with the repo version before deploying

### Pitfall 5: functions.php Getting Duplicate Hooks
**What goes wrong:** Running the functions.php append command twice, creating duplicate `fcg_set_fund_designation_early` function
**Why it happens:** The append method adds code to end of file; if run twice, PHP fatal error on duplicate function name
**How to avoid:** Before appending, grep staging functions.php to confirm the function isn't already there: `grep -n "fcg_set_fund_designation_early" ~/sites/frederickc2stg/wp-content/themes/community-foundation/functions.php`
